css结构问题一

1.margin-top , margin-bottom不能正常显示
有时会遇到外层中的子层使用margin-top不管用的情况，只需要在子层的前后加上div{height:0;overflow:hidden} 或者是让子层浮动一下或者是给父层加上边框； 子层在底部时，给子层加上margin-bottom：10px也不管用，也要给个清除属性的.clear{clear:both;font-size:0;line-height:0} 在底部下加个

2.div层中高度自适应问题
不能规定div层的高度： div{ min-height:200px; height：auto;!important; height:200px; overflow:visible; } 这样ie7,ff,ie6的自适应问题就解决了

3.div层中子层的居底部对齐问题
子层定位bottom：0；

4.div层中清除clear属性的一小部分应用
在div层中有很多子层，若是加上边框在ie7、ie6中或许会正常显示，但是在ff中可能 只会成一条线了，此时在最外层的后面加上或者设 .wrapfix:after{ content: "."; display: block; height: 0; clear: both; visibility: hidden; }后在每个浮动外框调用wrapfix；

5.解决IE8下div移位、错位等兼容性问题
在head标签后面的第一句话加上就好了

6.IE6中高度不对的问题
在div中给定了高度为1px，其他浏览器正常显示，而在ie6显示的高度不对，这时给样式表加font-size:0px;line-height:0px;

7.ff中margin-top有时不起作用的问题
在div_center里套个ul li时在ff中出问题，在div_top和div_center中多几个像素的空格，这时给ul设个display：inline-table;

8.IE6下，li:hover的兼容性问题
鼠标悬停效果 li:hover,div:hover等，是css2及以上版本才添加的，在IE6中只兼容a:hover，要使用li:hover或div：hover，需引用一个文件使其兼容，它就是csshover.htc 放在网站的根目录下，并在css文件中，加上
body
{
behavior:url("...csshover.htc");
}
